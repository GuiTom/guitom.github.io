1.为什么block的属性修饰符要用copy关键字?（什么时候需要对 block 进行 copy 操作？）

Block在没有使用外部变量时，内存存在全局区，然而，当Block在使用外部变量的时候，内存是存在于栈区，当Block copy之后，是存在堆区的。存在于栈区的特点是对象随时有可能被销毁，一旦销毁在调用的时候，就会造成系统的崩溃。所以Block要用copy关键字。

2.下面这段代码会的打印结果是什么？

- (void)viewDidLoad {

    [super viewDidLoad];

    // Do any additional setup after loading the view, typically from a nib.

    __block int mutiplier = 10;

    _blk = ^int(int num){

        return num*mutiplier;

    };

    mutiplier = 6;

    [self executeBlock];

    

}

-(void)executeBlock{

    int result = _blk(4);

    NSLog(@"%d",result);

}

答案：24

解析:用__block 修饰符修饰的 mutiplier 变量实际上被转换成了一个包含 mutiliper 变量的结构体,如下:

struct __Block_byref_mutiplier_0 {

  void *__isa;

__Block_byref_mutiplier_0 *__forwarding;

 int __flags;

 int __size;

 int mutiplier;

};

无论是 block内部还是外部，对mutiplier 变量的访问实际上是通过指向 mutiplier 的一个指针来实现的，内部和外部访问的指针指向的都是同一个结构体，

 (mutiplier.__forwarding->mutiplier) = 6;

在 block 执行的时候，其内部访问的那个 mutiliper 值其实被外部改写成了6，

因此，结果是24
